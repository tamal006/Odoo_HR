The gap

Odoo approvals happen blind.

Odoo's own docs note leave visibility is role-gated and that "poorly managed leave policies lead to understaffing." A manager sees a request — not the second-order consequence of granting it.

Odoo today

A calendar of who's out.

You approve, then find out three teammates overlap and the sprint week is uncovered. No forecast, no coverage math, no warning. The cost shows up later as burnout and missed deadlines.

With Impact Preview

The consequence, before you act.

The copilot re-simulates the 8-week capacity timeline as if the leave were approved, diffs it against now, and flags the week that crosses the floor — so you stagger it, not regret it.

Architecture

Three tiers. One Odoo truth.

The React app talks to an Express backend for CRUD and to a Python agent for intelligence. Both read the same Odoo — the backend over XML-RPC, the agent over the Model Context Protocol.
